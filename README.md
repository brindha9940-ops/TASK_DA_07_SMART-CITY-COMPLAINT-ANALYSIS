Smart City Complaint Analysis

Problem Statement

Analyze citizen complaints to identify recurring civic issues, evaluate service efficiency, and support data-driven decision-making for smart city management.

Dataset Description

The dataset contains 2000 citizen complaints with details such as complaint category, zone, department, priority, status, resolution delay, and citizen satisfaction score.

KPIs

- Total Complaints
- Resolved Complaints
- Resolution Rate
- Average Resolution Time
- High Priority Complaints
- Citizen Satisfaction Score

Tools Used

- Python
- Pandas
- NumPy
- Matplotlib
- Seaborn
- Google Colab

Key Findings

- Certain complaint categories occur more frequently.
- Some zones experience recurring issues.
- Resolution delays reduce citizen satisfaction.
- Department performance varies across services.

Recommendations

- Focus on complaint hotspot zones.
- Reduce resolution delays.
- Improve handling of high-priority complaints.
- Monitor department performance regularly.

Future Scope

- Real-time complaint monitoring
- GIS-based mapping
- AI-powered complaint classification
- Predictive analytics
